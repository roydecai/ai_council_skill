# 异常降级

## 目标

覆盖调用层失败场景，保证 AI Council 在部分 agent 失效时仍能稳定推进，并明确什么时候可以降级继续、什么时候必须中止。

## 适用范围

本文件只处理调用层异常，不处理观点质量异常。输出过短、偏题、矛盾等内容质量问题仍按 `agent-config.md` 中的主持检查规则处理。

## MVP 降级底线

- 最少可继续开会人数：`3`
- MVP 最低阵容：皮皮 + Kimi + GPT Codex
- 若活跃人数少于 `3`，会议不得继续推进到正式决议

说明：

- 皮皮必须始终在线，否则会议直接中止。
- Kimi 与 GPT Codex 是 MVP 必备成员；若原本还有 MiniMax / Gemini，可先从可选成员开始降级移除。

## 异常分类

### 1. 单 agent 超时

判定建议：

- 单轮等待超过 5 分钟仍无响应
- 或明显超过本轮其他 agent 平均耗时 3 倍以上

处理流程：

1. 主持人标记该 agent 为 `timeout`。
2. 进行一次轻量重试，重试只发送：
   - 当前状态摘要
   - 本轮必须回答的问题
   - 明确的超时提醒
3. 若重试仍失败：
   - 若当前仍有至少 3 名活跃成员，降级继续
   - 若不足 3 名，终止会议并输出中止说明

### 2. 单 agent 调用失败

典型场景：

- API 错误
- CLI 退出非零
- session 失效
- 连接被中断

处理流程：

1. 主持人记录错误类型与失败轮次。
2. 优先尝试同模型同会话重连。
3. 若会话不可恢复，允许重新拉起该 agent，但必须广播最新状态对象。
4. 若重拉仍失败，则将该 agent 标记为 `failed`，进入人数判断。

### 3. 部分返回

典型场景：

- 4 人会议只回来 3 人
- 5 人会议只回来 3 或 4 人

处理原则：

- 若返回人数不少于 `3`，可以继续，但必须在轮次总结中显式写明“本轮缺席成员”。
- 缺席成员不计入本轮共识判断。
- 如果缺席成员在下轮恢复，只能基于最新状态对象补位，不允许要求其他人重讲整轮内容。

### 4. 主持人上下文损坏

典型场景：

- 主持人会话重置
- 主持人忘记前序轮次

处理流程：

1. 立即停止发起下一轮。
2. 从 `state-protocol.md` 定义的状态对象恢复。
3. 若状态对象也缺失，则会议降为“重新开会”，不得假装延续原讨论。

## 降级顺序

当原始阵容大于 3 人时，降级顺序建议如下：

1. 先移除可选成员：Gemini
2. 再移除可选成员：MiniMax
3. 保留 MVP 核心阵容：皮皮 + Kimi + GPT Codex

不得降级掉皮皮。
不得在 MVP 模式下只剩 2 人还继续产出正式决议。

## 降级后的轮次调整

### 从 4 人降为 3 人

- `light` 模式：维持 2 轮，不额外加轮
- `standard` 模式：默认维持 3 轮
- `deep` 模式：若缺席发生在第 1 或第 2 轮，可把最大轮次从 5 收缩为 4；若已进入第 4 轮，通常不再补轮

调整原则：

- 不因为少 1 人就强行重开
- 只在该成员承担关键证据角色且无人可替代时，追加 1 轮澄清

### 从 5 人降为 3 人

- 立即切回 MVP 阵容
- 主持人重写当轮问题，收缩为“关键决策 / 高重要性分歧 / 未决证据”三项
- 后续默认不再进入额外发散轮

## 降级后主持人必须做的事

1. 更新状态对象中的 `agents` 和 `meeting.status`。
2. 在轮次总结中写明：
   - 谁缺席
   - 缺席原因
   - 是否重试
   - 是否影响结论可信度
3. 重写下一轮任务，避免继续依赖失效成员独占的信息。
4. 在最终决议中加入“降级说明”。

## 最终决议中的降级说明建议

```md
## 降级说明
- 异常类型：Kimi 第 2 轮超时
- 处理动作：重试 1 次失败，会议降级为 3 人继续
- 影响评估：不影响主结论，但减少了资料整理视角
```

## 中止条件

满足任一条件时，会议必须中止：

- 皮皮不可用
- 活跃成员少于 3
- 关键状态对象丢失且无法恢复
- 剩余成员都无法提供支撑主结论所需证据

## 与状态协议的联动

降级时，状态对象至少要更新以下字段：

```json
{
  "meeting": {
    "status": "degraded"
  },
  "agents": [
    {
      "id": "minimax",
      "status": "timeout"
    }
  ],
  "next_step": {
    "action": "degrade_and_continue",
    "reason": "活跃人数仍满足最少 3 人"
  }
}
```

## 主持人一句话准则

先保住 3 人 MVP，再压缩轮次与问题范围；人数不够就停，不做假完整决议。
